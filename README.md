# README – CS‑320 Portfolio Reflections

**How can I ensure that my code, program, or software is functional and secure?**

I make my code functional and secure by combining disciplined development habits with thorough testing. Writing unit tests early helps me verify that each method behaves the way the requirements describe, and boundary and negative tests help me catch unexpected behavior before it becomes a real issue. I also focus on clarity and predictability in my code so it’s easier to maintain and harder to misuse. Security comes from validating inputs, avoiding assumptions, and thinking about how the system might fail or be misused. Testing isn’t just about proving that code works—it’s about proving that it fails safely.


**How do I interpret user needs and incorporate them into a program?**

I start by breaking down user requirements into specific, testable behaviors. Instead of assuming what the user wants, I translate each requirement into conditions the software must meet. This helps me stay grounded in the customer’s expectations rather than my own assumptions. Writing tests based on requirements also forces me to think through edge cases and real‑world scenarios. When I understand the “why” behind a requirement, it becomes easier to design features that actually solve the user’s problem.


**How do I approach designing software?**

I approach software design by thinking in small, reliable components. Each class or method should have a clear purpose, and the interactions between components should be predictable. Testing plays a big role in my design process because it helps me identify where complexity might cause issues. I try to design with maintainability in mind—clean structure, minimal repetition, and clear logic. When I slow down and think about how the system behaves as a whole, I end up with code that’s easier to test, easier to extend, and easier to trust.
